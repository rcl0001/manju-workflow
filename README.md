# 漫剧自动生成工作流
完全按照自定义链路开发的小说转漫剧全链路自动化项目。
## 🔗 工作流链路
```
输入小说 → 剧本生成Agent → 角色信息 → 角色三视图生成Agent → 角色参考图
                                 ↓
                          环境信息 → 环境参考图生成Agent → 场景参考图
                                 ↓
                          剧本内容 → 视频提示词生成Agent → 视频提示词
                                 ↓
          角色参考图 + 场景参考图 + 视频提示词 → 视频生成Agent → 最终漫剧视频
```
## 📁 项目结构
```
manju-workflow/
├── agents/                # 5个核心Agent
│   ├── 1_剧本生成Agent.js
│   ├── 2_角色三视图生成Agent.js
│   ├── 3_环境参考图生成Agent.js
│   ├── 4_视频提示词生成Agent.js
│   └── 5_视频生成Agent.js
├── assets/                # 所有输出产物
│   ├── chars/             # 生成的角色三视图
│   ├── scenes/            # 生成的场景参考图
│   └── videos/            # 生成的视频片段和最终漫剧
├── utils/                 # 通用工具函数
│   └── fileUtils.js       # 文件读写工具
├── main.js                # 主工作流入口
├── package.json           # 依赖配置
└── README.md              # 使用说明
```
## 🚀 3步快速跑通
### 1. 安装依赖
```bash
npm install
```
### 2. 配置你的API
修改每个Agent里的`callLLM`、`callImageGenModel`、`callVideoGenModel`函数，替换成你自己的大模型、生图模型、视频生成模型的实际API调用。
### 3. 运行工作流
修改`main.js`里的`testNovel`为你的小说内容，然后运行：
```bash
npm start
```
## ✨ 特性
- 完全匹配你要求的输出格式，剧本输出和示例完全一致
- 所有中间产物自动保存到assets目录，方便调试和迭代
- 每个Agent完全解耦，修改某个环节的逻辑不影响其他模块
- 支持自定义提示词和生成参数，方便调优
## 🔧 迭代优化
1. 先跑通测试用例，验证每个环节的输出格式是否符合要求
2. 调整每个Agent里的Prompt，优化生成质量
3. 增加错误重试逻辑，提升稳定性
4. 加入质检环节，自动修复不合格的生成结果
